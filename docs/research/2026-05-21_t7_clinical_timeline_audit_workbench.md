Date: 2026-05-21

Branch: `codex/clinical-timeline-audit`

Base: `origin/main@d7037c7`

## Objective

Turn the Phase B manual label audit from prose into an inspectable, reproducible
artifact. The project already exports seizure-centered audit rows and active
review queues; this task adds a static timeline workbench and companion review
sheet so a human reviewer can inspect source-aligned windows without editing
code.

This task does not make clinical verdicts and does not create citable benchmark
numbers. It creates the audit interface and storage format that Gate C needs.

## Implementation

- Added `src/reports/clinical_audit_workbench.py`.
- Added CLI `scripts/build_clinical_audit_workbench.py`.
- Exported the workbench helpers from `src/reports/__init__.py`.
- Extended allowed review decisions with `UNCERTAIN`, which is valid but still
  blocks final pass validation.
- Added synthetic tests in `tests/test_clinical_audit_workbench.py`.

## Workbench Outputs

The CLI writes:

- `audit_workbench.html`
- `timeline_geometry.csv`
- `review_sheet.csv`
- `audit_workbench_summary.csv`
- `audit_workbench_report.json`
- `audit_workbench.md`

The HTML displays seizure-centered windows, SPH/SOP positive-zone geometry,
ictal intervals, optional postictal exclusion intervals, forecast positives,
exclusions, alarms, and reviewer data rows.

## Scientific Guardrails

- Reviewer decision fields remain blank; the workbench never infers clinical
  correctness.
- `PASS`, `FAIL`, `UNCERTAIN`, and `NEEDS_SOURCE_REVIEW` are accepted decision
  values, but only all-`PASS` sheets can pass validation.
- Citable output requires `gate_c_status=passed`.
- The Markdown and JSON exports carry `result_status`, `citation_status`, and
  `gate_c_status`.

## Remaining Limits

- The static HTML is a viewer; persistent reviewer decisions are stored in the
  CSV review sheet, not edited in-browser.
- Real Gate C closure still requires a clinician or qualified reviewer to fill
  the review sheet against source annotations.
- The viewer does not decode raw waveform files; it visualizes audit rows
  already generated by the preprocessing pipeline.

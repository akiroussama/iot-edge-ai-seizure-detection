# EpiBench Real Evidence Package Workorder

## Objective

Move the manuscript from protocol demonstration to Q1-ready empirical evidence by building real evidence packages for at least two public datasets.

## Minimum Required Before npj Submission

### Package A - Clinical EEG Detection

Preferred dataset:

- TUSZ if access and tooling are ready.

Alternative:

- CHB-MIT if the goal is a smaller demonstration with clearer operational control.

Track:

- `D`, seizure detection.

Required artefacts:

- `dataset_card.yaml`;
- `split_manifest.yaml`;
- `failure_trace.yaml`;
- `result_bundle.yaml`;
- event metrics report;
- claim eligibility report;
- naive leaderboard row;
- EpiBench claim-gated row;
- figure-ready per-patient distribution.

Scientific objective:

- show that EpiBench can wrap an EEG detection benchmark without replacing event-scoring standards.

Current status as of 2026-05-24:

- CHB-MIT metadata-based Track `D` package exists at `examples/epibench/chbmit_patient_independent_d/`.
- The package is generated by `scripts/epibench_build_chbmit_package.py` from official PhysioNet metadata and summary annotations.
- Certification result: `E2-PI`, `Dataset-T1`, no blocking failure.
- Epi-Score: `3.186` because the model is intentionally `always_negative`.
- Scientific boundary: this closes the patient-independent evidence-structure gap, but it does not provide a useful EEG detector. A waveform-based baseline remains required for a strong Q1 main result.

### Package B - Wearable Multimodal Detection

Preferred dataset:

- SeizeIT2.

Track:

- `D`, wearable seizure detection.

Required artefacts:

- full Dataset Evidence Card;
- MTS/DSI with wearable-specific stress items;
- split manifest preserving patient independence;
- missingness/artifact failure trace;
- baseline suite: always negative, rate-matched random, simple movement threshold, ECG/HR threshold if available, small model baseline;
- claim eligibility report.

Scientific objective:

- show that hospital/wearable evidence does not automatically imply home IoT generalization.

### Optional Package C - Longitudinal Forecasting

Candidate:

- My Seizure Gauge or another longitudinal wearable dataset where access and licensing permit.

Track:

- `F`, seizure forecasting.

Required artefacts:

- SPH/SOP definitions;
- right-censoring policy;
- time-in-warning and FAR/day;
- Brier score and ECE;
- patient-specific temporal split;
- claim report.

Scientific objective:

- show that EpiBench distinguishes forecasting from detection.

Current status as of 2026-05-24:

- MSG Gate C frozen forecasting package exists at `examples/epibench/msg_gate_c_frozen_f/`.
- The package is generated from the frozen benchmark manifest by `scripts/epibench_build_msg_gate_c_package.py`.
- Certification result: `E2-PD`, `Dataset-T2`, `Epi-Score 58.984`, no blocking failure.
- Scientific boundary: this is a submission-grade longitudinal forecasting evidence package, but it is not patient-independent and therefore cannot satisfy the required `E2-PI` operational package gate.
- Interpretation: Package C is useful for demonstrating track separation and fail-closed claim ceilings; it does not replace Package A or Package B.

## Baseline Suite

Each package must include baselines appropriate to the track:

- always negative;
- prevalence or rate-matched random;
- simple signal threshold;
- logistic regression or linear baseline;
- small CNN/GRU/TCN baseline only after trivial baselines are correct;
- patient-specific threshold if patient-dependent analysis is declared.

## Figures From Real Packages

Required:

1. Dataset evidence heatmap: MTS versus DSI.
2. Naive leaderboard.
3. Claim-gated leaderboard.
4. Failure heatmap.
5. Sensitivity versus FAR/day.
6. Per-patient distribution and worst patient.
7. Claim gate waterfall.

## Acceptance Criteria

The package is ready only if:

- all artefacts validate against schemas;
- CLI certification reproduces the reported claim;
- failures are visible;
- baselines are included;
- claims are conservative;
- a clinician or domain expert reviews label categories and overclaim language.

## Failure Conditions

Do not include a dataset in the main paper if:

- label provenance is unknown;
- split leakage cannot be audited;
- event denominators are unclear;
- preprocessing is not traceable;
- licensing prevents reproducible release;
- the paper would require unsupported clinical claims.

# EpiBench SzCORE Official Contract Report

Status: official scorer compatibility smoke test present.

## Scientific Purpose

A Q1 reviewer could reasonably reject EpiBench if it appeared to reinvent seizure event scoring while
SzCORE already exists as a seizure-community evaluation framework. The EpiBench position is therefore
explicit:

- SzCORE remains the event/sample scoring authority where it is applicable.
- EpiBench does not replace SzCORE event scoring.
- EpiBench imports official or SzCORE-compatible event metrics, preserves their provenance, and applies
  dataset evidence, split validity, failure sentinels, hardware evidence, and claim gates around them.

## Official Sources Used

- `szcore-evaluation==0.0.7`, PyPI package: https://pypi.org/project/szcore-evaluation/
- SzCORE framework page: https://epilepsybenchmarks.com/framework/
- SzCORE tools page: https://epilepsybenchmarks.com/tools/
- SzCORE paper record: https://arxiv.org/abs/2402.13005

## Repository Artefacts

- `examples/epibench/szcore_official_smoke/reference/`: minimal BIDS-like reference TSV.
- `examples/epibench/szcore_official_smoke/hypothesis/`: minimal BIDS-like hypothesis TSV.
- `examples/epibench/szcore_official_smoke/szcore_evaluation_result.json`: raw output generated by
  `python -m szcore_evaluation`.
- `examples/epibench/szcore_official_smoke/run_manifest.yaml`: package versions, command, contract, and
  claim boundary.
- `src/epibench/szcore_bridge.py`: maps `event_results` from the official JSON contract into EpiBench
  metric names.
- `tests/test_epibench_standard.py`: verifies that official `event_results.sensitivity`, `precision`,
  `f1`, and `fpRate` map into EpiBench event sensitivity, event precision, event F1, and false alarms per
  24 hours.

## Mapping Contract

| Official SzCORE field | EpiBench field |
| --- | --- |
| `event_results.sensitivity` | `event_sensitivity` |
| `event_results.precision` | `event_precision` |
| `event_results.f1` | `event_f1` |
| `event_results.fpRate` | `false_alarms_per_24h` |

The bridge writes:

- `external_event_scoring_mapped: true`;
- `external_event_scoring_relationship: MAP`;
- `external_event_scoring_official_contract: szcore-evaluation.evaluate_dataset:event_results`;
- `external_event_scoring_mapped_fields`: deterministic source-to-target field list.

## Acceptance Criterion

The A04 evidence is considered materially improved when a clean checkout can:

1. run the official `szcore-evaluation` smoke fixture;
2. map the raw official JSON shape into an EpiBench Result Bundle;
3. certify the mapped bundle without changing SzCORE scores;
4. preserve the statement that EpiBench is a claim-gating layer, not an event-scoring replacement.

## Remaining Boundary Before Submission

This smoke fixture is not a seizure-detection result. Before manuscript submission, at least one real
EEG detection package should preferably include a full official SzCORE run on its event outputs. Until
then, this artefact closes the reinvention risk at the API-contract level, not at the full external
benchmark-execution level.
